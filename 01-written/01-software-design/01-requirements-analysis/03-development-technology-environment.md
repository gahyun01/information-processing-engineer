# 💻 03. 개발 기술 환경

## 📖 개요

개발 기술 환경 분석은 **개발하고자 하는 소프트웨어가 운영될 환경을 파악하고, 적합한 기술 요소를 선정하기 위한 과정**이다.

운영체제(OS), 데이터베이스 관리 시스템(DBMS), 미들웨어, 웹 애플리케이션 서버(WAS) 등의 기술 환경을 분석하여 시스템 개발에 필요한 요구사항을 결정한다.

#### 🎯 목적

- 개발 시스템에 적합한 기술 환경을 선정한다.
- 기존 시스템과의 호환성을 확인한다.
- 시스템 성능과 안정성을 확보한다.
- 구축 및 유지보수 비용을 고려한다.

## 🏗️ 플랫폼 ( Platform )

### 📖 정의

플랫폼은 **애플리케이션을 구동시키는 데 필요한 하드웨어와 소프트웨어 환경**이다.

공급자와 사용자 등 여러 참여자가 서비스를 제공하고 이용할 수 있도록 구성된 기반 환경을 의미한다.

#### ✨ 특징

- 애플리케이션 실행에 필요한 환경을 제공한다.
- 동일 플랫폼에서는 소프트웨어 간 상호 호환이 가능하다.
- 여러 사용자가 참여하여 가치를 교환하는 환경을 제공한다.

<br/>

### 📊 플랫폼 성능 특성 분석

플랫폼 성능 분석은 **사용자가 서비스를 이용할 때 시스템 응답 속도와 처리 능력이 적절한지 판단하기 위한 과정**이다.

#### 📌 성능 측정 항목

> 📝 암기: ***경사응가***
>
> **경**과 시간 ( Turnaround Time )  
> **사**용률 ( Utilization )  
> **응**답 시간 ( Response Time )  
> **가**용성 ( Availability )

| 항목 | 설명 |
|---|---|
| 경과 시간 ( Turnaround Time ) | 작업 요청부터 처리가 완료될 때까지 걸리는 시간 |
| 사용률 ( Utilization ) | CPU, 메모리 등 자원의 사용 정도 |
| 응답 시간 ( Response Time ) | 요청 전달부터 응답 도착까지 걸리는 시간 |
| 가용성 ( Availability ) | 시스템이 정상적으로 사용 가능한 정도 |

## 🖥️ 운영체제 ( OS, Operating System )

### 📖 정의

운영체제는 **컴퓨터 하드웨어와 소프트웨어 자원을 관리하고, 사용자가 컴퓨터를 효율적으로 사용할 수 있도록 환경을 제공하는 시스템 소프트웨어**이다.

#### ✨ 역할

- 하드웨어 자원 관리
- 프로세스 관리
- 메모리 관리
- 파일 관리
- 사용자와 하드웨어 간 인터페이스 제공

<br/>

### 🔍 운영체제 선정 시 고려사항

운영체제는 다음 요소를 기준으로 분석한다.

#### 1. 가용성

- 장기간 운영 시 장애 발생 가능성
- 장애 발생 시 재가동 필요 여부
- 지속적인 패치 지원 여부

#### 2. 성능

- 대규모 사용자 요청 처리 가능 여부
- 대량 파일 처리 능력
- 지원 가능한 메모리 크기

#### 3. 기술 지원

- 공급 업체의 안정적인 지원
- 사용자 커뮤니티 활성화 여부
- 오픈소스 지원 여부

#### 4. 주변 기기 지원

- 설치 가능한 하드웨어
- 다양한 주변 기기 지원 여부

#### 5. 구축 비용

- 하드웨어 비용
- 라이선스 비용
- 유지관리 비용
- 총 소유 비용(TCO)

<br/>

### 💰 총 소유 비용 ( TCO, Total Cost of Ownership )

시스템 구축부터 운영까지 발생하는 모든 직·간접 비용을 의미한다.

#### 포함 비용

- 하드웨어 구매 비용
- 소프트웨어 구매 및 라이선스 비용
- 설치 비용
- 교육 비용
- 유지보수 비용
- 운영 비용

## 💾 데이터베이스 관리 시스템 ( DBMS )

### 📖 정의

DBMS ( Database Management System ) 는 **데이터베이스를 생성하고 저장 및 관리하기 위한 소프트웨어**이다.

기존 파일 시스템의 문제점인 **데이터 종속성과 데이터 중복성**을 해결하고, 여러 응용 프로그램이 데이터를 공유할 수 있도록 관리한다.

<br/>

### 🔍 DBMS 선정 시 고려사항

> 📝 암기: ***가성호기구***
>
> **가**용성  
> **성**능  
> **호**환성  
> **기**술 지원  
> **구**축 비용

| 항목 | 내용 |
|---|---|
| 가용성 | 장애 발생 가능성, 백업 및 복구, 이중화 지원 |
| 성능 | 대량 데이터 처리, 트랜잭션 처리, 튜닝 지원 |
| 호환성 | 운영체제, JDBC·ODBC 지원 여부 |
| 기술 지원 | 공급 업체 지원, 사용자 정보 공유, 오픈소스 여부 |
| 구축 비용 | 라이선스, 유지관리 비용, TCO |

## ⚙️ 미들웨어 ( Middleware )

### 📖 정의

미들웨어는 **운영체제와 응용 프로그램 사이에서 추가적인 서비스를 제공하는 소프트웨어**이다.

#### ✨ 역할

- 데이터 관리
- 통신 기능 제공
- 트랜잭션 관리
- 세션 관리

<br/><br/>

<hr/>

<br/><br/>

## 🌐 웹 애플리케이션 서버 ( WAS )

### 📖 정의

WAS ( Web Application Server ) 는 **웹 서버와 달리 동적인 콘텐츠를 처리하고 데이터 접근, 세션 관리, 트랜잭션 관리 등의 기능을 제공하는 미들웨어**이다.

#### ✨ 특징

- 동적 웹 서비스 처리
- 데이터베이스 연동
- 세션 관리
- 트랜잭션 관리

<br/>

### 🔍 WAS 선정 시 고려사항

| 항목 | 내용 |
|---|---|
| 가용성 | 장애 발생 가능성, 이중화 지원 |
| 성능 | 대규모 트랜잭션 처리, GC 설정 지원 |
| 기술 지원 | 제조업체 지원, 오픈소스 여부 |
| 구축 비용 | 라이선스 비용, 유지관리 비용, TCO |

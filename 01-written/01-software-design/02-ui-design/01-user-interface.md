# 💻 01. 사용자 인터페이스 ( UI, User Interface )

## 📖 인터페이스 ( Interface )

### 📖 정의

인터페이스는 **서로 다른 두 시스템이나 소프트웨어를 연결하여 상호작용할 수 있도록 하는 접속 장치 또는 연결 부분**이다.

#### ✨ 특징

- 서로 다른 시스템이나 구성 요소 간의 정보를 교환하는 접점이다.
- 하드웨어와 소프트웨어, 사용자와 시스템 사이에서도 사용된다.
- 물리적 인터페이스와 논리적 인터페이스로 구분할 수 있다.

## 🖥️ 사용자 인터페이스 ( UI )

### 📖 정의

사용자 인터페이스 ( UI ) 는 **사용자와 시스템이 서로 정보를 주고받으며 상호작용할 수 있도록 하는 물리적·논리적 매개체**이다.

좁은 의미에서는 사용자가 직접 접하는 화면이나 입력 장치를 의미하며, 넓은 의미에서는 사용자와 시스템 사이의 모든 상호작용 환경을 의미한다.


#### ✨ 특징

- 사용자와 시스템 간의 원활한 의사소통을 지원한다.
- 사용자의 편의성과 가독성을 높여 작업 시간을 단축한다.
- 최소한의 노력으로 원하는 결과를 얻을 수 있도록 한다.
- 사용자 중심으로 설계되어 상호작용을 향상시킨다.
- 수행 결과의 오류를 줄이고 피드백을 제공한다.
- 소프트웨어 영역 중 변경이 가장 많이 발생하는 요소이다.
- UI 설계를 위해서는 시스템의 기능과 소프트웨어 아키텍처를 충분히 이해해야 한다.

> 💡 초기의 UI는 단순한 입출력 중심이었지만, 현재는 사용자 경험 ( UX ) 을 고려한 정보 전달 중심으로 발전하였다.

<br/>

### 🎯 사용자 인터페이스의 역할

- 사용자와 시스템 간 상호작용을 지원한다.
- 사용자가 작업을 쉽고 빠르게 수행하도록 한다.
- 정보 제공자와 정보 이용자 사이의 매개 역할을 수행한다.
- 사용자의 작업 오류를 최소화한다.

<br/>

### ⚙️ 사용자 인터페이스의 기본 기능

- 입력(Input) : 사용자의 명령이나 데이터를 시스템에 전달한다.
- 출력(Output) : 처리 결과를 사용자에게 제공한다.
- 피드백(Feedback) : 처리 결과나 오류 상태를 사용자에게 알려준다.

<br/>

### 🧩 사용자 인터페이스 구성 분야

- 정보 제공 및 전달을 위한 물리적 제어 분야
- 콘텐츠의 표현과 화면 구성 분야
- 사용자의 편리성과 사용성을 높이는 기능 분야

<br/><br/>

### 📱 사용자 인터페이스 유형

> 📝 암기 : **CG NO**

| 유형 | 설명 |
|---|---|
| CLI ( Command Line Interface ) | 명령어를 텍스트로 입력하여 사용하는 인터페이스 |
| GUI ( Graphical User Interface ) | 아이콘, 메뉴 등 그래픽 요소를 이용하는 인터페이스 |
| NUI ( Natural User Interface ) | 터치, 음성, 제스처 등 사람의 자연스러운 행동을 이용하는 인터페이스 |
| OUI ( Organic User Interface ) | 현실의 사물과 상호작용하는 인터페이스 ( IoT, VR, AR 등 ) |
| VUI ( Voice User Interface ) | 사람의 음성을 이용하여 제어하는 인터페이스 |

<br/><br/>

### 📐 사용자 인터페이스 설계 원칙

> 📝 암기 : **직유학유**

| 원칙 | 설명 |
|---|---|
| 직관성 ( Intuitiveness ) | 누구나 쉽게 이해하고 사용할 수 있어야 한다. |
| 유효성 ( Effectiveness ) | 사용자의 목표를 정확하고 완전하게 달성할 수 있어야 한다. |
| 학습성 ( Learnability ) | 초보자와 숙련자 모두 쉽게 배우고 사용할 수 있어야 한다. |
| 유연성 ( Flexibility ) | 다양한 사용자 요구를 수용하고 실수를 최소화해야 한다. |

<br/>

### 🔄 인터랙션 ( Interaction )

#### 📖 정의

인터랙션은 **사용자와 시스템이 입력과 출력을 통해 서로 상호작용하는 과정**이다.

#### ✨ 특징

- 사용자의 입력에 대해 시스템이 적절한 결과와 피드백을 제공한다.
- UI의 사용성과 사용자 경험(UX)에 직접적인 영향을 미친다.

## 🛠️ UI 설계 지침

### 👤 사용자 중심

- 사용자가 쉽게 이해하고 사용할 수 있도록 설계한다.
- 심미성보다 사용성을 우선 고려한다.

### ⚡ 효율성

- 버튼과 조작 방법을 쉽게 기억하고 빠르게 사용할 수 있도록 한다.

### ✨ 단순성

- 조작 방법을 단순화하여 사용자의 인지 부담을 줄인다.

### 🔮 결과 예측 가능

- 기능만 보아도 수행 결과를 예측할 수 있어야 한다.

### 👀 가시성

- 주요 기능을 메인 화면에 배치하여 쉽게 사용할 수 있도록 한다.

### 📏 표준화

- 기능 구조와 디자인을 일관성 있게 구성한다.

### ♿ 접근성

- 다양한 사용자 계층이 사용할 수 있도록 설계한다.

### 🎯 명확성

- 사용자가 기능을 쉽게 이해할 수 있도록 표현한다.

### 👍 사용성 (Usability)

- 쉽고 편리하며 효율적으로 사용할 수 있어야 한다.
- UI 설계 시 가장 중요하게 고려하는 요소이다.

### 🔄 일관성

- 버튼, 메뉴, 화면 구성을 일관되게 유지한다.

### 🎨 심미성

- 글꼴, 색상, 그래픽 등을 적절히 활용하여 가독성을 높인다.

### ⚠️ 오류 처리

- 오류를 쉽게 인지할 수 있도록 한다.
- 이해하기 쉬운 오류 메시지를 제공한다.
- 오류 복구 방법을 함께 안내한다.
- 오류 발생 시 적절한 피드백을 제공한다.

## ⚙️ 사용자 인터페이스 개발 시스템의 기능

UI 개발 시스템은 다음 기능을 제공해야 한다.

- 사용자 입력 검증
- 오류 처리 및 오류 메시지 제공
- 도움말(Help) 및 프롬프트(Prompt) 제공

## 🖼️ UI 화면 구성 요소

### 📖 정의

UI 화면은 사용자가 시스템과 상호작용하기 위해 사용하는 입력 요소, 페이지 요소, 팝업, Alert 등의 화면 요소로 구성된다.

### 📦 입력(Control) 요소

| 구성 요소 | 설명 |
|---|---|
| 텍스트 박스 | 텍스트를 입력하는 입력창 |
| 콤보 박스 | 목록 중 하나를 선택하는 입력 요소 |
| 토글 버튼 | 두 가지 상태를 전환하는 버튼 |
| 라디오 버튼 | 여러 항목 중 하나만 선택 |
| 체크 박스 | 여러 항목을 동시에 선택 |

<br/>

### 📄 페이지(Page) 요소

- 그리드(Grid)
- 아이콘
- 탭(Tab)
- 스텝(Step)
- 페이지 이동
- 스크롤
- 정보 영역

<br/>

### 🪟 팝업(Popup) 요소

- 윈도 팝업(Window Popup)
- 레이어 팝업(Layer Popup)

<br/>

### 🚨 Alert 요소

- 정보 누락 / 오류 알림
- 삭제 / 수정 확인
- 작업 완료 알림
- 업무 안내 메시지

## 🤏 UI 제스처 ( Gesture )

### 📖 정의

UI 제스처는 **터치 기반 환경에서 손가락의 움직임을 이용하여 시스템을 조작하는 입력 방식**이다.

| 제스처 | 설명 |
|---|---|
| 탭(Tap) | 화면을 한 번 가볍게 터치 |
| 더블 탭(Double Tap) | 화면을 두 번 연속 터치 |
| 프레스(Press) | 화면을 길게 누름 |
| 플릭(Flick) | 빠르게 밀어 넘김 |
| 스와이프(Swipe) | 손가락을 떼지 않고 일정 방향으로 이동 |
| 팬(Pan) | 화면을 끌어서 이동 |
| 드래그(Drag) | 대상을 끌어 이동한 후 놓음 |
| 핀치(Pinch) | 두 손가락을 모으거나 벌림 |
| 로테이트(Rotate) | 두 손가락으로 회전 |

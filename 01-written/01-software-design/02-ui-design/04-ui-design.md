# 💻 04. UI 상세 설계

## 📖 UI 상세 설계

### 📖 정의

UI 상세 설계는 **UI 설계서를 바탕으로 실제 구현을 위해 화면 구성, 화면 간 흐름, 인터랙션, 예외 처리 등을 구체적으로 설계하는 과정**이다.

UI 흐름 설계에서 작성한 UI 설계서를 보완하여 실제 개발에 필요한 상세 내용을 완성한다.

#### ✨ 특징

- UI 설계서를 기반으로 수행한다.
- 화면별 세부 기능과 동작을 정의한다.
- 화면 간 이동 흐름과 인터랙션을 구체화한다.
- 다양한 예외 상황과 처리 방법을 설계한다.

> 💡 UI 상세 설계에서 가장 먼저 수행하는 작업은 **UI 시나리오 문서 작성**이다.

## 📄 UI 시나리오 문서

### 📖 정의

UI 시나리오 문서는 **사용자의 목표 달성을 위해 화면의 기능, 화면 간 흐름, 인터랙션, 예외 처리 등을 순차적으로 기술한 문서**이다.

UI 설계자가 작성하며, 디자이너와 개발자가 실제 화면을 구현하는 기준 문서로 활용한다.

#### ✨ 특징

- 화면의 기능과 구조를 정의한다.
- 화면 간 이동 흐름을 표현한다.
- 인터랙션과 예외 처리를 포함한다.
- 사용자 중심의 작업 흐름을 순서대로 기술한다.

<br/>

### 📋 UI 설계서와의 관계

UI 설계서는 **UI 흐름 설계와 UI 상세 설계에서 모두 작성**된다.

| 구분 | 설명 |
|---|---|
| UI 흐름 설계 | UI 설계서의 기본 구조를 작성 |
| UI 상세 설계 | UI 설계서를 보완·수정하여 최종 완성 |

<br/>

<hr/>

<br/>

## ✍️ UI 시나리오 문서 작성 원칙

UI 시나리오 문서는 개발자가 UI의 기능과 동작을 쉽게 이해할 수 있도록 작성해야 한다.

일반적으로 **계층 구조** 또는 **플로우차트** 형태로 작성한다.

### 📌 작성 내용

- 공통 UI 요소와 인터랙션 규칙 정의
- 대표 화면의 레이아웃과 기능 정의
- 화면 간 이동 흐름 정의
- 조건, 분기, 반복 ( Loop ) 정의
- 예외 상황과 처리 방법 정의
- 기능별 상세 시나리오 정의
- UI 시나리오 작성 규칙을 정의

<br/>

<hr/>

<br/>

## 📐 UI 시나리오 문서의 일반 규칙

| 항목 | 설명 |
|---|---|
| 주요 키의 위치와 기능 | 공통 버튼의 위치와 기능을 일관성 있게 정의 |
| 공통 UI 요소 | 체크박스, 라디오 버튼, 텍스트 박스 등의 사용 규칙 정의 |
| 기본 화면 레이아웃 | Title, OK/Back 버튼, Soft Key, Functional Buttons 등의 공통 화면 구성 정의 |
| 기본 인터랙션 규칙 | 화면 이동, 터치 제스처 등의 공통 동작 정의 |
| 공통 단위 태스크 흐름 | 검색, 삭제 등 공통 기능의 처리 흐름 정의 |
| 케이스 문서 | 예외 상황에서 시스템의 동작 정의 |

## ✅ UI 시나리오 문서의 요건

### 📌 완전성 ( Completeness )

- 기능이 누락되지 않도록 상세하게 작성한다.
- 시스템 기능보다 **사용자의 작업( Task )** 중심으로 기술한다.

<br/>

### 📌 일관성 ( Consistency )

- 서비스 목표, UI 스타일, 요구사항 등이 일관성을 유지해야 한다.

<br/>

### 📌 가독성 ( Readability )

- 표준 템플릿을 사용한다.
- 문서 버전 ( v1.0, v2.0 등 ) 을 관리한다.
- 목차, 들여쓰기, 줄 간격 등을 일정하게 유지한다.
- 문서 간 하이퍼링크 등을 활용하여 참조성을 높인다.

<br/>

### 📌 수정 용이성 ( Modifiability )

- 변경 및 유지보수가 쉽도록 작성한다.

<br/>

### 📌 추적 용이성 ( Traceability )

- 변경 이력과 변경 사유를 쉽게 확인할 수 있도록 관리한다.

## 🎯 UI 상세 설계의 기대 효과

- 요구사항에 대한 오류 감소
- 의사소통 오류 최소화
- 개발 과정의 재작업 감소
- 불필요한 기능 제거
- 개발 비용 절감
- 개발 기간 단축

<br/>

<hr/>

<br/>

## 💡 용어 정리

| 용어 | 설명 |
|---|---|
| 인터랙션 ( Interaction ) | 사용자와 시스템 사이의 상호작용 과정 |
| 인터랙션 디자이너 | 사용자의 행동과 시스템의 반응을 설계하는 사람 |
| 템플릿 ( Template ) | 화면의 기본 레이아웃과 문서 형식 |

## 📝 UI 상세 설계 핵심 정리

| 구분 | 핵심 내용 |
|---|---|
| UI 상세 설계 | UI 설계서를 기반으로 화면을 구체적으로 설계 |
| UI 시나리오 문서 | 화면 기능, 흐름, 인터랙션을 정의하는 문서 |
| UI 설계서 | UI 흐름 설계와 UI 상세 설계에서 모두 작성 |
| 작성 원칙 | 화면 구성, 인터랙션, 예외 처리 등을 구체적으로 기술 |
| 일반 규칙 | 공통 UI 요소, 화면 레이아웃, 인터랙션 규칙 정의 |
| 문서 요건 | 완전성, 일관성, 가독성, 수정 용이성, 추적 용이성 |
| 기대 효과 | 오류 감소, 재작업 감소, 비용 절감, 개발 기간 단축 |
